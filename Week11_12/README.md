# 可微渲染实验：从球体到奶牛 —— 基于 PyTorch3D 的网格形状与纹理联合优化

## 一、实验简介

本实验基于 PyTorch3D 实现了一套完整的**可微渲染（Differentiable Rendering）**流水线：以一个细分球体为初始网格，仅依靠多视角的二维监督信号（剪影图 / RGB 图），通过梯度反传逐步优化三维网格的顶点坐标与顶点颜色，最终重建出目标奶牛模型的形状与外观。

实验分为两个阶段：

- **阶段一（基础任务）**：仅用 20 个视角的剪影图作为监督，优化顶点偏移量，重建奶牛的几何形状。
- **阶段二（选做 / 联合纹理优化）**：在阶段一形状的基础上，新增可学习的顶点颜色参数，改用 `SoftPhongShader` 渲染 RGB 图像，同时拟合形状与外观。

环境：阿里云 ModelScope（魔搭社区）免费 GPU 算力，PyTorch3D 通过 Gitee 镜像源码编译安装。

---

## 二、实验原理回顾

| 模块 | 作用 |
|---|---|
| **软光栅化（SoftSilhouetteShader）** | 用 sigmoid 函数将像素到三角形边缘的距离映射为平滑的占用概率 $A(d)=\text{sigmoid}(d/\sigma)$，避免硬光栅化在边界处梯度为 0 的问题，使顶点在像素外部也能获得有效梯度。 |
| **拉普拉斯平滑（Laplacian Smoothing）** | 约束相邻顶点位置一致，抑制局部尖锐凸起。 |
| **边长惩罚（Edge Loss）** | 惩罚过长/过短的边，防止三角形被严重拉伸。 |
| **法线一致性（Normal Consistency）** | 约束相邻面法线方向接近，保持表面连续光滑。 |

总损失：

$$L_{total} = L_{sil}\;(+\,L_{rgb}) + w_{lap}L_{lap} + w_{edge}L_{edge} + w_{normal}L_{normal}$$

若仅靠图像差异驱动优化、不加正则化，顶点会为了迎合单一视角投影而疯狂交叉重叠，最终陷入"刺猬"状的局部最优——这正是本实验要验证和规避的核心问题。

## 三、阶段一：剪影拟合（球体 → 奶牛形状）

初始球体仅靠剪影监督，配合三项正则化，经过 300 轮迭代逐渐"捏"出奶牛轮廓。

**优化中期（Epoch 80，总 Loss 0.1000）**：球体刚开始向外凸出头部和四肢雏形，整体仍接近球状，细节尚未分化。

![阶段一中期](images/stage1_epoch080.png)

**优化收敛（Epoch 299，总 Loss 0.0179，剪影误差 0.0141）**：头、耳、四肢、躯干轮廓清晰可辨，与 Ground Truth 剪影高度吻合。

![阶段一最终结果](images/stage1_epoch299_final.png)

可以看到，剪影误差从初始的较高水平稳步下降至 0.0141，预测剪影的轮廓细节（耳朵尖角、四肢分叉）已经能够准确复现目标剪影的形状特征。

---

## 四、阶段二（选做）：联合纹理优化（形状 + RGB 颜色）

在阶段一形状基础上，新增顶点颜色参数 `verts_rgb_joint`，改用 `SoftPhongShader` 同时渲染并监督 RGB 图像，总损失同时包含剪影 MSE、RGB MSE 与三项正则化。

**优化中期（Iter 50，总 Loss 0.0111，RGB Loss 0.0015）**：形状已基本就位，但躯干衔接处仍可见明显的"接缝感"瑕疵，颜色拟合也尚未稳定。

![阶段二中期](images/stage2_iter050.png)

**优化收敛（Iter 499，总 Loss 0.0083，剪影 0.0011，RGB 0.0007）**：预测 RGB 图像与目标灰色调几乎一致，剪影边缘也更加干净平滑，接缝瑕疵基本消失。

![阶段二最终结果](images/stage2_iter499_final.png)

> 说明：本实验中目标网格的外观被初始化为占位灰白色（未使用 obj 自带贴图），因此颜色变化空间有限，RGB Loss 能够收敛到很低水平（0.0007）符合预期；该模块的重点在于验证"形状 + 外观联合优化"的可行性与梯度流动的正确性，而非追求绚丽的纹理还原。

## 五、定量与多视角验证

### 5.1 收敛曲线

两阶段的总 Loss 及各子项 Loss 均呈现平滑下降并最终收敛到平台期，说明优化过程稳定、未出现震荡或发散：

![Loss收敛曲线](images/loss_curves.png)

- 阶段一：总 Loss 从约 0.23 平滑下降至 0.018 左右，剪影 Loss 与总 Loss 曲线几乎重合（因为阶段一只有剪影监督）。
- 阶段二：总 Loss 在前 50 个 epoch 内快速从 0.055 降至 0.01 附近，随后进入长尾平台期，500 epoch 时已基本收敛在 0.0083，说明 500 epoch 的设置足够，未出现欠拟合。

### 5.2 多视角泛化验证

为了排除"仅在训练用的 20 个视角上拟合良好、其他角度形变畸形"的过拟合可能，额外选取 8 个**未参与训练**的新视角（不同方位角 + 20° 俯仰角）重新渲染最终网格：

![多视角验证](images/multiview_validation.png)

可以看到，在所有未见过的视角下，模型依然保持完整合理的奶牛轮廓（四肢、躯干、头部比例协调），没有出现局部塌陷或穿插畸变，说明优化得到的是真实的三维几何结构，而非仅在训练视角上"作弊"的二维投影伪解。

### 5.3 量化指标：Chamfer Distance 与拓扑质量

![Chamfer Distance与拓扑质量指标](images/chamfer_topology.png)

| 指标 | 数值 | 含义 |
|---|---|---|
| Chamfer Distance（优化网格 vs 目标网格） | **0.001437** | 采样点云间双向最近邻距离，数值很小，说明重建形状与真实奶牛网格高度接近 |
| 法线一致性损失 | 0.016053 | 越接近 0 表面越光滑，当前值处于合理范围 |
| 边长惩罚 | 0.002610 | 三角形边长分布规整，未出现严重拉伸 |
| 拉普拉斯平滑损失 | 0.003778 | 局部曲率平滑，未见明显尖刺凸起 |

以上指标共同表明：最终重建网格不仅在二维投影层面（剪影/RGB）与目标高度吻合，在三维几何层面（点云距离、表面光滑性、拓扑规整性）同样达到了较高质量，验证了正则化项在防止网格崩坏方面的有效性。

## 六、实验收获与思考

1. **梯度消失问题的本质**：硬光栅化下，像素归属是离散的 0/1 判断，三角形边界处的导数处处为 0，顶点完全无法感知"该往哪个方向移动才能让投影更接近目标"。Soft Rasterization 通过 sigmoid 近似将这个阶跃函数变得连续可导，是整个可微渲染管线能够工作的根本前提。

2. **正则化不是可有可无的"锦上添花"，而是优化能否收敛到合理解的决定性因素**。仅靠图像 Loss 驱动，顶点完全可能为了迎合某一视角的剪影/RGB 投影而产生自相交、尖刺等违反物理直觉的局部最优解——这本质上是一个欠约束的反问题（多个三维形状可以投影出相同的二维图像），正则化项相当于引入了关于"自然物体表面应当光滑、规整"的先验知识，把解空间收缩到合理范围内。

3. **形状先验证 vs 外观先验证的耦合关系**：阶段二联合优化中，剪影 Loss 在前期下降很快说明形状信息相对外观信息更容易从多视角监督中恢复（剪影对几何边界更敏感），而 RGB Loss 的收敛对网格的局部表面法线方向（影响 Phong 光照效果）更为敏感，这也是阶段二选择在阶段一收敛形状的基础上继续优化、而非从零开始的原因——避免形状和外观两个相互耦合的变量在优化早期互相干扰、增加收敛难度。

4. **多视角验证与量化指标的必要性**：仅凭训练时展示的几张对比图容易"看起来很完美"，但这些视角本身就是监督信号的来源，存在过拟合风险。引入未参与训练的新视角渲染、以及 Chamfer Distance 等与训练目标解耦的量化指标，才能更客观地评估三维重建质量，这是本次实验中体会最深的一点。

## 七、工程实践踩坑记录

实验过程中遇到的主要工程问题是 **PyTorch3D 在 ModelScope 上的安装**：通过 `git clone` Gitee 镜像源码时反复出现 TLS 连接中断（HTTP/2 智能协议在弱网环境下不稳定）。最终解决方案是放弃 git 智能协议，改用 `wget` 直接下载仓库压缩包（单次 HTTP GET，更抗网络抖动），配合 `--tries` 自动重试机制成功获取源码后本地编译安装。这一调整使安装成功率显著提升，也是本实验中容易被忽视、但实际耗时最长的环节之一。

## 九、运行方式

1. 在与 `cow.obj` 同一目录下打开 `cow_fitting.ipynb`。
2. 依次执行环境安装、目标渲染、阶段一优化、阶段二优化（选做）、结果验证各部分 cell。
3. 中间结果模型可下载到本地，使用 **MeshLab** 打开查看三维效果（`.obj` 查看几何，`.ply` 查看带颜色的最终效果）。
